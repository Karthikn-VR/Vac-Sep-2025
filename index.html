Get involved — how to contribute

Choose a file, fix a bug, add a small useful tool or example — then open a Pull Request. This page includes quick commands and a PR template to make contributing easy.

Create PR Open Issue

Quick start (copy & paste)

git clone https://github.com/your-org/vac-sep-2025.git
cd vac-sep-2025
# create a branch
git checkout -b feat/your-name-short-description
# edit files, then:
git add .
git commit -m "feat: short description (your-name)"
git push origin feat/your-name-short-description
# open a PR from the branch on GitHub

PR checklist

  • Follow the coding style used in the project.
  • Keep changes small and focussed.
  • Include a short description and testing notes in the PR body.
  • Add yourself to CONTRIBUTORS.md if you added a new example.

PR template (example)

## What I changed
- Short description of change

## Why
- Reason and benefit

## Notes for reviewers
- Anything reviewers should know (manual steps, configs)

## Checklist
- [ ] My change follows project style
- [ ] I tested my change locally
